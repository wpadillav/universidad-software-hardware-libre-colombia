# 3. Modelo Educativo: Aprender Haciendo, Certificar Demostrando

## 3.1 Filosofia Pedagogica

El modelo adopta una sintesis de tres tradiciones pedagogicas probadas:

- **Construccionismo** de Seymour Papert (aprender construyendo artefactos)
- **Aprendizaje entre pares** (peer learning) de P2PU
- **Pedagogia de proyectos** de hackerspaces y makerspaces

La premisa central: la mejor forma de aprender sobre software libre es
contribuyendo a un proyecto de software libre; la mejor forma de aprender
sobre hardware libre es disenando y construyendo hardware libre.

---

## 3.2 Rutas de Aprendizaje Modulares

En lugar de carreras rigidas, la universidad ofrece rutas modulares que los
participantes recorren a su propio ritmo. Cada modulo culmina en un producto
demostrable.

| Ruta | Modulos | Producto Final |
|---|---|---|
| Fundamentos del Software Libre | Historia, licencias GPL/MIT/Apache, etica digital, GNU/Linux | Contribucion aceptada en proyecto de software libre |
| Desarrollo de Software Libre | Git/GitHub, Python/JS/C, metodologias agiles, testing | Aplicacion o libreria publicada con licencia libre |
| Hardware Libre y Diseno | Electronica, Arduino/Raspberry Pi, KiCad, impresion 3D | Dispositivo hardware certificado por OSHWA |
| Soberania Tecnologica | Servidores propios, redes mesh, DNS, correo autonomo | Servidor o red comunitaria operativa |
| Datos Abiertos y Conocimiento Libre | OER, Wikidata, visualizacion, periodismo de datos | Conjunto de datos publicado o articulo en Wikipedia |
| Fabricacion Digital Libre | FreeCAD, CAM, CNC, materiales sustentables | Objeto fabricado con documentacion abierta |

## 3.3 Sistema de Certificacion por Competencias

Tres niveles de certificacion:

1. **Micro-credentials** (Open Badges): insignias digitales por habilidades
   especificas, emitidas automaticamente al completar cada modulo.

2. **Competencias Laborales Certificadas** (modelo SENA): evaluacion humana
   que recoge evidencias de desempeno, producto y conocimiento.

3. **Portafolios de Competencias Avanzadas**: documentos comprensivos,
   verificables y portables, que funcionan como un curriculum viviente.

El modelo de UoPeople logro acreditacion WASC en 2025 combinando peer
assessment, OER y voluntarios academicos, demostrando que esta aproximacion
puede alcanzar reconocimiento formal.
